响应式，链式，函数式编程思想 ReactiveCocoa

##链式编程思想

###作用以及好处

  • 思路

    链式编程思想，方法的返回值是block block中的返回值必须是对象本身

  • 好处

    增加可读性 典型框架 masonry为了 高聚合低耦合的思想设计的框架，方便维护查找问题。

###代码实现

  • 增加一个类的分类本文列举 NSObject+cacuate 增加了一个计算的功能

    #import "NSObject+cacuate.h"
@implementation NSObject (cacuate)
-(int)caculate:(void (^)(Caculate *))block{
    Caculate *m = [[Caculate alloc] init];
    block(m);
    return m.result;
    }
    @end

  • 增加一个专门处理计算的类

    .h文件
#import <Foundation/Foundation.h>
@interface Caculate : NSObject
@property (nonatomic,assign)int  result;
- (Caculate *(^)(int))add;
@end

.m文件
#import "Caculate.h"
@implementation Caculate
-(Caculate * (^)(int))add{
    return ^(int value){
        _result = _result + value;
        return self;
    };
}
@end

  • 使用

    int result = [self caculate:^(Caculate * m) {
m.add(5).add(5);
}];
NSLog(@"%d",result);

##函数式编程思想

###作用以及好处

  • 思路

    往方法中传入block 方法中嵌套block调用，把代码聚合起来管理,每个方法必须有返回值（本身对象）
    把函数或者block当做参数（需要操作的值）block返回值（操作的结果）

  • 代表

    ReactiveCocoa框架 （如何使用该框架详情请见http://www.taotieming.com）
    ###代码实现

    @interface XYSCaculate : NSObject
@property (nonatomic,assign)int  result;
-(instancetype)xysCaculate:(int(^)(int))block;        

 .m文件
#import "XYSCaculate.h"
@implementation XYSCaculate
-(instancetype)xysCaculate:(int (^)(int))block{
    _result = block(_result);
    return self;
}
@end

  • 使用

    HSSCaculate *o = [[XYSCaculate alloc] init];
o.result = 6;
int t =[[o xysCaculate:^int(int result) {
    result = result + 9;
    return result;
}] result];
NSLog(@"%d",t);
